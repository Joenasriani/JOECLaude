# Contributing to JOECLaude

Thank you for your interest in contributing to JOECLaude! We welcome contributions from the community.

## How to Contribute

### 1. Fork the Repository

Click the "Fork" button at the top right of the [repository page](https://github.com/Joenasriani/JOECLaude).

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/JOECLaude.git
cd JOECLaude
```

### 3. Create a Branch

Create a new branch for your feature or bug fix:

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Your Changes

- Write clear, readable code
- Follow existing code style and conventions
- Add or update documentation as needed
- Test your changes thoroughly

### 5. Commit Your Changes

Write clear and descriptive commit messages:

```bash
git add .
git commit -m "Add: description of your change"
```

### 6. Push and Create a Pull Request

```bash
git push origin feature/your-feature-name
```

Then go to the original repository and create a Pull Request.

## Pull Request Process

1. Ensure your PR description clearly describes the problem and solution.
2. Reference any related issues using `#issue-number`.
3. Make sure all CI checks pass.
4. Request a review from a maintainer.
5. Be responsive to feedback and make requested changes promptly.

## Types of Contributions

- **Bug Fixes:** Found a bug? Submit a fix!
- **Features:** Have an idea? Open an issue first to discuss, then submit a PR.
- **Documentation:** Improvements to docs are always welcome.
- **Tests:** Help us improve test coverage.

## Code of Conduct

By participating in this project, you agree to:

- Be respectful and inclusive
- Provide constructive feedback
- Accept constructive criticism gracefully
- Focus on what is best for the community and the project

## Questions?

If you have questions about contributing, feel free to open an issue with the `question` label.

Thank you for contributing! 🎉
