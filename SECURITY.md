# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| latest  | :white_check_mark: |

## Reporting a Vulnerability

We take the security of JOECLaude seriously. If you discover a security vulnerability, please follow these steps:

1. **Do NOT open a public issue.** Security vulnerabilities should be reported privately.
2. **Email us** at [joenasr@gmail.com](mailto:joenasr@gmail.com) with:
   - A description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact of the vulnerability
   - Any suggested fixes (if available)
3. **Alternatively**, use [GitHub Security Advisories](https://github.com/Joenasriani/JOECLaude/security/advisories/new) to report the vulnerability privately.

## Response Timeline

- **Acknowledgement:** We will acknowledge your report within **48 hours**.
- **Assessment:** We will assess the vulnerability and determine its severity within **5 business days**.
- **Resolution:** We aim to resolve critical vulnerabilities within **30 days** of the initial report.

## Disclosure Policy

- We follow a **coordinated disclosure** process.
- We will credit reporters (unless they prefer to remain anonymous) once the vulnerability is resolved.
- We ask that you give us a reasonable amount of time to address the issue before any public disclosure.

## Scope

This security policy applies to:
- All code in the `main` branch
- All officially released versions
- The analytics module and any future modules

Thank you for helping keep JOECLaude and its users safe!
